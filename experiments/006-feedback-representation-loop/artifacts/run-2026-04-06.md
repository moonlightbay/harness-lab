## Observations

- Requirement-only feedback preserves the invariant in prose, but it does not clearly say what failed or what to rerun next.
- A human review comment is highly actionable, but it is weaker as a reusable harness primitive because it does not separate actual state, expected state, and pass condition.
- A generic failing check names the problem, but it still leaves too much repair planning implicit.
- A remediation-oriented failing check is the only package that fully describes diagnosis, action, rerun command, and success condition.

## Verdict

Use failing checks for stable invariants, and make those checks remediation-oriented by default.
