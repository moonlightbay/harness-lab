## Scenario details

- Nested branch context:

  - changed files: `README.md`, `src/feature_a.py`, `src/feature_b.py`
  - README markers: `task_a`, `task_b`
- Worktree A context:
  - branch: `task-a`
  - changed files: `README.md`, `src/feature_a.py`
  - README markers: `task_a`
- Worktree B context:
  - branch: `task-b`
  - changed files: `README.md`, `src/feature_b.py`
  - README markers: `task_b`

## Observations

- The nested branch forces both tasks into one dirty review surface, so neither task can be shipped or discarded without manual split work.
- The worktree pattern keeps both tasks reviewable and restartable as independent contexts even though both touch `README.md`.
