# Git And Sync

- Use branch-per-task when multiple lines of work are active.
- Use a worktree for any parallel or risky wave.
- Pause every 5 commits for a branch-level review and plan correction.
- Treat upstream or mainline changes as a translation task.
- Do not mix translation work with new training design work in one wave.
- If a sync changes the baseline, record the old and new baseline explicitly in the run log.
