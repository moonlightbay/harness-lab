# Initial Garbage-Collection Checklist

Use this after each experiment or documentation change.

- Remove or archive stale experiment artifacts that are no longer referenced by a result summary.
- Update `docs/knowledge/` when a conclusion changes; stale docs count as harness failures.
- Update `docs/plans/experiment-plan.md` and `docs/plans/experiment-log.md` together when experiment status changes.
- Collapse duplicate helper logic once the second copy appears unless the duplication is deliberate and documented.
- Delete placeholders that no longer map to planned work.
- Mark deferred follow-ups explicitly instead of leaving orphaned TODO fragments.
